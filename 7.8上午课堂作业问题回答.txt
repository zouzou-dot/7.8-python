思考题一：为什么 NumPy 的矢量化运算比 Python 循环高效？
NumPy 的矢量化运算之所以远优于 Python 原生循环，根本原因在于两者执行机制的本质差异。
Python 作为动态解释型语言，其循环语句每次迭代都需要进行类型检查、临时对象创建和内存管理，这些重复性开销随着数据规模增大而急剧膨胀，形成明显的性能瓶颈。NumPy 则将核心运算交给 C 语言实现，绕过了 Python 解释器的逐行执行，同时充分利用现代 CPU 的 SIMD 指令集实现数据级并行，加之 NumPy 数组在内存中连续存储带来的高缓存命中率，使得相同规模的计算任务速度可提升数十甚至上百倍。因此在实际数据处理中，优先采用矢量化操作、避免显式循环是提升代码效率的关键策略。

思考题二：reshape 和 resize 有什么区别？
reshape 与 resize 虽然都用于改变数组形状，但在多个关键方面存在本质区别。首先是是否修改原数组，reshape 返回新对象而保持原数组不变，遵循不可变性原则；resize 则直接在原数组上修改，属于原地操作。其次是元素数量的灵活性，reshape 要求新形状的元素总数必须与原来严格一致，否则报错；resize 则允许改变元素总量，不足时补零、多余时丢弃。此外，reshape 通常与原数组共享内存，执行速度快且开销小；resize 则涉及数据复制，代价相对更高。实践中若非必须就地修改，优先使用 reshape 更为安全高效。
